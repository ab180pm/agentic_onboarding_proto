Initialize SDK

SDK initialization methods vary depending on the system architecture. For SceneDelegate Lifecycle or AppDelegate Lifecycle, refer to the method for AppDelegate. For SwiftUI Lifecycle, refer to the method for SwiftUI.

Call the Airbridge.initializeSDK function at the very top of AppDelegate's application(_:didFinishLaunchingWithOptions:) function.

AppDelegate.swift 
import UIKit
import Airbridge

@main
class AppDelegate: UIResponder, UIApplicationDelegate {
    func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
    ) -> Bool {
        let option = AirbridgeOptionBuilder(name: "YOUR_APP_NAME", token: "YOUR_APP_SDK_TOKEN")
            .build()
        Airbridge.initializeSDK(option: option)
        return true
    }
}

Attention: Make sure that the Airbridge.initializeSDK function is called within the application(_:didFinishLaunchingWithOptions:) of the AppDelegate for proper functionality.

Call the [Airbridge initializeSDK] function at the very top of AppDelegate's application:didFinishLaunchingWithOptions: function.

AppDelegate.m 
#import <Airbridge/Airbridge.h>

@implementation AppDelegate

- (BOOL)application:(UIApplication *)application
    didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {

    AirbridgeOption *option = [[AirbridgeOptionBuilder alloc]
        initWithName:@"YOUR_APP_NAME"
        token:@"YOUR_APP_SDK_TOKEN"];

    return YES;
}

@end

3
Configure ATT Prompt

The IDFA can only be collected if users consent to data tracking via the App Tracking Transparency (ATT) prompt.

Event collection should be delayed until the user allows tracking. If the install event is collected before the user allows tracking through the ATT prompt, the install event data will lack an identifier, making performance measurement difficult.

1

Prepare the text you will use in the ATT prompt.

2

Enter the prepared text in the NSUserTrackingUsageDescription key of the Info.plist file.

Navigate to [YOUR_PROJECT] > [Info] > [Custom iOS Target Properties] in Xcode. Add "Privacy - Tracking Usage Description" key and enter the ATT prompt text as value.

3

Set the time for displaying the ATT prompt on the app.

The ATTrackingManager.requestTrackingAuthorization function should be called at the desired time you want to display the ATT prompt.

ViewController.swift 
import AppTrackingTransparency
...
ATTrackingManager.requestTrackingAuthorization { _ in }

The ATT prompt can be displayed as soon as the app is launched.

AppDelegate.swift 
import UIKit
import Airbridge
import AppTrackingTransparency

@main
class AppDelegate: UIResponder, UIApplicationDelegate {
    var observer: Any?

    func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
    ) -> Bool {
        let option = AirbridgeOptionBuilder(name: "YOUR_APP_NAME", token: "YOUR_APP_SDK_TOKEN")
            .build()
        Airbridge.initializeSDK(option: option)

        observer = NotificationCenter.default.addObserver(
            forName: UIApplication.didBecomeActiveNotification,
            object: nil,
            queue: nil
        ) { [weak self] _ in
            if #available(iOS 14, *) {
                ATTrackingManager.requestTrackingAuthorization { _ in }
            }
            if let observer = self?.observer {
                NotificationCenter.default.removeObserver(observer)
            }
        }

        return true
    }
}
4

Set a sufficient delay before event collection. The setAutoDetermineTrackingAuthorizationTimeout function can be used. The default setting is 30 seconds and can be set to up to 3600 seconds (1 hour).

AppDelegate.swift 
import Airbridge
...
let option = AirbridgeOptionBuilder(name: "YOUR_APP_NAME",
                                    token: "YOUR_APP_SDK_TOKEN")
    .setAutoDetermineTrackingAuthorizationTimeout(second: 30)
    .build()
Airbridge.initializeSDK(option: option)

Attention: Set a sufficient delay time to collect the app install event. If the SDK collects install events before users can allow tracking on the ATT prompt, the IDFA won't be collected along with the install events.

import UIKit
import Airbridge

class SceneDelegate: UIResponder, UIWindowSceneDelegate {

    // Deep link handler
    func handleAirbridgeDeeplink(url: URL) {
        // Navigate to specific screen based on URL
    }

    // Cold start with Universal Link
    func scene(_ scene: UIScene, willConnectTo session: UISceneSession,
               options connectionOptions: UIScene.ConnectionOptions) {
        // Handle userActivities
        if let userActivity = connectionOptions.userActivities.first {
            Airbridge.trackDeeplink(userActivity) { url in
                self.handleAirbridgeDeeplink(url: url)
            }
        }
        // Handle urlContexts
        if let urlContext = connectionOptions.urlContexts.first {
            Airbridge.trackDeeplink(urlContext.url) { url in
                self.handleAirbridgeDeeplink(url: url)
            }
        }
    }

    // Scheme deep link (app in background)
    func scene(_ scene: UIScene,
               openURLContexts URLContexts: Set<UIOpenURLContext>) {
        guard let urlContext = URLContexts.first else { return }
        Airbridge.trackDeeplink(urlContext.url) { url in
            self.handleAirbridgeDeeplink(url: url)
        }
    }

    // Universal Link (app in background)
    func scene(_ scene: UIScene,
               continue userActivity: NSUserActivity) {
        Airbridge.trackDeeplink(userActivity) { url in
            self.handleAirbridgeDeeplink(url: url)
        }
    }
}
